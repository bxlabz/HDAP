Nginx Reverse Proxy

Handles incoming HTTP requests and forwards them to the Flask application.

  • Rate limiting (10 req/sec with burst of 20)
  • Request size limit (10MB max)
  • Proxy timeout handling (300s read, 75s connect)

Flask Application Server

Core Python backend handling authentication, API endpoints, and business logic.

  • User authentication with Flask-Login
  • Session management
  • REST API endpoints
  • Static file serving
Python 3.11 Flask 3.0

Geocoding Service

Converts street addresses to geographic coordinates using OpenStreetMap Nominatim.

  • Address variation handling

Route Optimization Algorithm

1
Proximity Clustering

Locations are grouped into clusters based on geographic proximity. Starting from the location nearest to the depot, each cluster grows by adding the nearest unassigned location until reaching the maximum stops limit.

2
Nearest Neighbor TSP

Within each cluster, the route is optimized using the Nearest Neighbor heuristic. Starting from the depot, the algorithm visits the closest unvisited location repeatedly until all stops are covered.

3
Distance Calculation

Distances are calculated using the geodesic (Vincenty) formula for accuracy on Earth's curved surface. A 1.35x multiplier approximates actual road distances vs. straight-line distances.

4
Return to Depot

Each optimized route includes a return leg to the starting depot, creating a complete round-trip circuit for delivery drivers.
